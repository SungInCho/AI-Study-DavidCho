# Monte Carlo Methods

## Overview

Monte Carlo methods estimate value functions by **sampling episodes** and averaging returns. Unlike DP, they require no knowledge of environment dynamics — they are fully **model-free**.

---

## Key Concept: Return

The return $G_t$ is the discounted sum of future rewards from time $t$:

$$G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots = \sum_{k=0}^{T-t-1} \gamma^k R_{t+k+1}$$

MC estimates value functions by averaging sampled returns:

$$V(s) \approx \mathbb{E}[G_t | S_t = s]$$

---

## First-visit vs Every-visit

| | First-visit | Every-visit |
|--|-------------|-------------|
| Updates | Only first occurrence of $s$ per episode | Every occurrence of $s$ |
| Bias | Unbiased | Biased (early episodes) |
| Variance | Lower | Higher |
| Convergence | Both converge to $V^\pi$ with sufficient episodes |

---

## Algorithms

### 1. MC Prediction

Estimate $V^\pi$ and $Q^\pi$ under a fixed policy using sampled episodes.

**Incremental mean update:**
$$V(s) \leftarrow V(s) + \frac{1}{N(s)}(G - V(s))$$

```python
mc = MonteCarlo(env)
episodes = mc.gen_episodes(policy, num=100)
V, Q = mc.first_visit_pred(policy)
V, Q = mc.every_visit_pred(policy)
```

---

### 2. Exploring Starts (ES) Control

Guarantee exploration by starting each episode at a **random (state, action) pair**.

**Algorithm:**
```
Initialize Q, π randomly
Repeat:
    Generate episode starting from random (s, a)
    For each (s, a) in episode:
        Update Q(s,a) with G
    π(s) ← argmax_a Q(s,a)
```

```python
policy, Q = mc.ES_control(max_iter=10000)
```

**Limitation:** Cannot be applied when we cannot choose arbitrary starting states.

---

### 3. On-policy MC Control (ε-greedy)

Use ε-greedy policy to ensure exploration without requiring exploring starts.

$$\pi(a|s) = \begin{cases} 1 - \varepsilon + \frac{\varepsilon}{|A|} & a = \arg\max Q \\ \frac{\varepsilon}{|A|} & \text{otherwise} \end{cases}$$

Converges to **ε-soft optimal policy**, not true $\pi^*$.  
Use $\varepsilon = 1/t$ decay to approach $\pi^*$ over time.

```python
policy, Q = mc.onpolicy(max_iter=10000, epsilon=0.1)
```

---

### 4. Off-policy MC Prediction (Weighted IS)

Learn $V^\pi$ (target policy) from episodes generated by $b$ (behavior policy) using **Weighted Importance Sampling**.

**Importance Sampling Ratio:**
$$\rho_t = \prod_{k=t}^{T} \frac{\pi(a_k|s_k)}{b(a_k|s_k)}$$

**Weighted IS update (incremental):**
$$C(s,a) \leftarrow C(s,a) + W$$
$$Q(s,a) \leftarrow Q(s,a) + \frac{W}{C(s,a)}(G - Q(s,a))$$

**Coverage condition:** $b(a|s) > 0$ whenever $\pi(a|s) > 0$

```python
Q = mc.offpolicy_pred(target_policy, max_iter=10000)
```

**Limitation:** If target policy is greedy, $W=0$ whenever a non-greedy action is taken → early episode termination → data inefficiency.

---

## Limitations of MC

- Must wait until **episode ends** to update
- High variance due to sampling
- Off-policy MC is data-inefficient (W=0 problem)
- These limitations motivated TD methods
